<a name="cl-25"></a><span class="k">\section*</span><span class="nb">{</span>Background<span class="nb">}</span>
<a name="cl-26"></a>The demand for easy, highly scalable, parallel computing is growing daily as
<a name="cl-27"></a>many-core systems become prevalent. The Message Passing Interface (MPI) is a
<a name="cl-28"></a>widely used and globally accepted model for expressing parallelism. The success
<a name="cl-29"></a>of MPI stems from its portability, completeness and performance i.r.t. massively
<a name="cl-30"></a>parallel programs, amongst others. However, MPI is generally not considered
<a name="cl-31"></a>simple, nor easy to program, particularly as it is most commonly implemented in
<a name="cl-32"></a>C or FORTRAN. This is a problem since many potential users are not skilled
<a name="cl-33"></a>programmers, but rather researchers who are capable of expressing their
<a name="cl-34"></a>algorithms in a deterministic logical framework.
<a name="cl-35"></a><span class="k">\\</span>
<a name="cl-36"></a>
<a name="cl-37"></a>The difficulty involved in ensuring the correctness and maintainability of C has
<a name="cl-38"></a>lead to the rise of higher level languages, such as Python. Python is a
<a name="cl-39"></a>language that originated in academia, and has large and growing popularity in
<a name="cl-40"></a>this group where it is often used for prototyping solutions to scientific
<a name="cl-41"></a>problems. Amongst Pythons many advantages are a strong typing system, a highly
<a name="cl-42"></a>readable syntax, existence on many platforms, and it is quite mature.
<a name="cl-43"></a><span class="k">\\</span>
<a name="cl-44"></a>
<a name="cl-45"></a>Python has no built-in support for MPI, but several other projects have grafted
<a name="cl-46"></a>MPI functionality onto Python, with varying degrees of completeness. At present,
<a name="cl-47"></a>there seems to be no native Python MPI implementations openly available.
<a name="cl-48"></a>
<a name="cl-49"></a><span class="c">%1 mpi is a globally accepted standard</span>
<a name="cl-50"></a><span class="c">%2 python is mature and widely used in scientific computing circles.</span>
<a name="cl-51"></a><span class="c">%3 python is cross platform</span>
<a name="cl-52"></a><span class="c">%4 many existing implementations in C and fortran, other languages often wrappers.</span>
<a name="cl-53"></a><span class="c">%5 both mpi and python widely used in academia and education.</span>
<a name="cl-54"></a>
<a name="cl-55"></a><span class="k">\section*</span><span class="nb">{</span>Motivation<span class="nb">}</span>
<a name="cl-56"></a>Coupling Python with MPI would seem to have several distinct advantages. 
<a name="cl-57"></a><span class="k">\begin</span><span class="nb">{</span>enumerate<span class="nb">}</span>
<a name="cl-58"></a>	<span class="k">\item</span> The advanced type system alleviates the complexities of MPI types as
<a name="cl-59"></a>Python types can be easily inferred at runtime. 
<a name="cl-60"></a>	<span class="k">\item</span> Python is standardized across platforms and is either already
<a name="cl-61"></a>installed, or very easy to install, on almost any major platform of interest to
<a name="cl-62"></a>HPC users. Python is also highly modular and easy to extend.
<a name="cl-63"></a>	<span class="k">\item</span> Implementing MPI in native Python means that there is no need to
<a name="cl-64"></a>maintain a separate native MPI installation, and by extension no need to keep
<a name="cl-65"></a>specific versions of Python, the MPI wrapper and the MPI library around to
<a name="cl-66"></a>maintain binary compatibility.
<a name="cl-67"></a>	<span class="k">\item</span> The low bar to entry to Python means that it is highly useful as a
<a name="cl-68"></a>prototyping language, and educators can focus on teaching central concepts of
<a name="cl-69"></a>concurrency, without the clutter of boilerplate code so typical to e.g. C.
<a name="cl-70"></a><span class="k">\end</span><span class="nb">{</span>enumerate<span class="nb">}</span>
<a name="cl-71"></a>
<a name="cl-72"></a><span class="c">%1 ease of installation</span>
<a name="cl-73"></a><span class="c">%1.5 crossplatform.</span>
<a name="cl-74"></a><span class="c">%2 problems with current wrappers</span>
<a name="cl-75"></a><span class="c">%3 python highly accessible to non computer scientist researchers.</span>
<a name="cl-76"></a><span class="c">%4 education.</span>
<a name="cl-77"></a><span class="c">%5 MPI types easily handled in python.</span>
<a name="cl-78"></a>
